## What's Working

1. **The CSS tooltip system is surgical.** [data-tip]::after on panel row icons solves a real problem: previously these required hover + browser title timeout. The CSS solution is instant, on-brand, accessible via focus-visible.
2. **Empty state teaches, not lectures.** Brief icon + heading + description + two specific CTAs (Go to Units / Go to Panels). Tradespeople on a fresh job now know exactly what to do next.

## Priority Issues

### [P2] No autosave status indicator

**Why it matters:** Tradespeople work on quotes mid-site-visit on a tablet. The app autosaves to localStorage on every keystroke — but there's no visual confirmation. Users have no way to distinguish "it's saved" from "it's about to be lost."

**Fix:** Add a small status chip near the job header — `● Saved` in text-muted that flashes to `● Saving…` in amber for ~1 second during writes. One debounced state change; big trust payoff.

**Suggested command:** /impeccable craft autosave indicator

---

### [P2] Unit quantity undo gap

**Why it matters:** Panel row changes can be reverted (↩ button) — but changing a room's unit quantity or material override has no undo path. A wrong material assignment silently distorts the quote total.

**Fix:** Mirror the _panelSnapshots pattern for room unit quantities. Snapshot quantities at each renderRoomView() call; add a per-section "Revert quantities" button.

**Suggested command:** /impeccable craft unit quantity revert

---

### [P2] Keyboard shortcut discoverability

**Why it matters:** Alt+1–5 and Alt+R are implemented but invisible. Power users must discover them by accident or hovering tab titles.

**Fix:** Wire ? (when not in a text field) to open a compact keyboard shortcut legend modal. Or add a one-line hint to the app footer.

**Suggested command:** /impeccable craft keyboard shortcut legend

---

### [P3] Light theme color drift

**Why it matters:** Detector flagged #edeae4, #f5f2ee, #f5e8d0, #4a3f30, and #1d6f42 (Excel green) as undocumented. These are real light-theme colors — not one-offs.

**Fix:** Add a light-theme block to DESIGN.md with named tokens. Purely a DESIGN.md edit.

**Suggested command:** /impeccable document (update pass)
