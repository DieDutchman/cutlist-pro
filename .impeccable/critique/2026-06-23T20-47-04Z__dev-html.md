## Overall Impression

**31/40 — Good.** This is a meaningful step up from the last run. The P1 violations are gone, the accessibility sweep is done, and the panel row revert is a real UX improvement for a pricing-critical tool. The design system is now internally consistent with no violations against its own rules.

## What's Working

1. **Design system now violation-free.** Heuristic 4 (Consistency) and 8 (Aesthetic) both hit 4/4. Every component now speaks the same language — amber focus rings, ghost buttons, icon-button accessibility, no side stripes.
2. **Panel revert is a genuine UX improvement.** The ↩ button appearing only when a field changes is well-scoped — not intrusive, but there when needed. For a pricing tool, preventing accidental material changes from silently distorting a quote is trust-building.
3. **Edge-code legend eliminates a recall dependency.** L1/L2/W1/W2 are now defined inline, collapsible, state-persisted. Users no longer need to find the info button or remember the abbreviations.
4. **Brand coherence and contrast ratios.** Both remain excellent. Dark theme contrast unchanged (13:1 primary text, 5:1 muted).
5. **Price change history log.** Still the standout domain-specific feature. Earns trust from tradespeople who rely on accurate numbers.

## Priority Issues

### [P2] Help and Documentation — still at 2/4

**What**: aria-labels are in place and the edge legend is live. But there is still no tooltip system for complex features, no onboarding, and no empty-state guidance beyond the Panels tab.

**Why it matters**: New tradespeople (non-software-native) hit feature gaps silently — they don't know what the ⚙ Special Services button does without clicking it. The quote Scenarios feature (the standout power feature) has no inline explanation of what A/B/C means to a client.

**Fix**: Add short `title`-equivalent visible descriptors to the three panel row buttons on first use (or at narrow widths where touch is primary). Add a one-liner "A/B/C options let clients compare price points" hint to the Scenarios strip.

**Suggested command**: `/impeccable onboard`

---

### [P2] Flexibility and Efficiency — no keyboard shortcuts

**What**: The entire app is mouse-only. Add Panel, switch tabs, duplicate a panel row — all require a mouse click. The tab bar has Alt+1–5 labels in the `title` attributes but no actual keyboard shortcut handlers.

**Why it matters**: Power users (Alex persona) doing 10+ quotes a week are slowed by repetitive mouse workflows. Tab navigation keyboard shortcuts are already described in the UI but not implemented.

**Fix**: Wire up the Alt+1–5 tab shortcuts that are already documented in the title attributes. Add a single keyboard shortcut for "Add Panel row" (e.g. Alt+A or Ctrl+Enter when focused in the panels view).

**Suggested command**: `/impeccable harden`

---

### [P2] "More" overflow tab discoverability

**What**: Several features live behind the "More" dropdown. There is no visual indicator that additional tabs exist beyond what's visible. New users don't know to look.

**Why it matters**: First-timers (Jordan persona) exploring the app will never find features like the Sheet Optimizer, P&L, or Supplier Pricing unless told about them.

**Fix**: Add a subtle dot or count indicator to the "More" button when overflow tabs exist ("More ▾ +4"). Or surface the two most-used overflow tabs as pinnable.

**Suggested command**: `/impeccable clarify`

---

### [P2] Backend error UX unknown

**What**: Supabase errors (network failures, auth expiry, save conflicts) are handled somewhere in the JS, but the user-facing error messages weren't audited. "Something went wrong" with no guidance is the common failure.

**Why it matters**: A tradesperson mid-quote who hits a network error and sees a cryptic message will lose confidence in the tool. In a pricing-critical context, this is a trust-breaker.

**Fix**: Audit all `catch` blocks and Supabase error paths; ensure errors surface a plain-language message with a recoverable action ("Connection lost — your work is saved locally, reconnecting…").

**Suggested command**: `/impeccable harden`

- Panel revert ↩ is per-row but there is no "revert all panels" for a full job reset.
