## Priority Issues

### [P2] No session-level undo for panel dimension/edging changes

**Why it matters:** Typing the wrong length or width is the most common data-entry mistake. panelRevert() restores last-saved state — which after an autosave cycle means the wrong value is the baseline. No "undo last edit" for a numeric field.

**Fix:** On panelField() focus, snapshot the current value. On blur with a changed value, offer a one-deep undo: extend _panelSnapshots to take a snapshot before each panelField() commit so panelRevert() always undoes the last field change.

**Suggested command:** /impeccable harden

---

### [P2] "More" overflow features still require a click to discover

**Why it matters:** The +N badge tells users there are N more things but not what they are. A new user still has no idea whether "More" hides critical or rarely-used features.

**Fix:** On first More menu open (localStorage flag), briefly show a one-shot tooltip listing the tab names. Or: add a popover-style preview on hover/focus showing tab names without requiring a click.

**Suggested command:** /impeccable onboard
