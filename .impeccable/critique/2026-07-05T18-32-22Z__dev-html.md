### Overall Impression

Still 37/40 — no regressions from the major feature additions (per-scenario state, drawer runner clearance, duplicate job, cutlist FROM chips). Two small gaps remain at P3: FROM chips look passive at rest (no visual invite to click), and switching scenarios shows no success signal to users who may expect only prices to change.

### What's Working

**Runner clearance live info line** — "Box outer width = 450 − 27 = 423mm" shown immediately under the preset buttons is textbook progressive disclosure. The tradesperson never does mental math; the formula is visible and recalculates on any change. This pattern earns trust through accuracy.

**Scenario hint copy** — "clones this option including its unit quantities, extras, tops and custom panels — great for presenting Option A / B / C alternatives" is precisely the right pitch for SA tradespeople. Plain language, real use case, not a feature description. Would pass the "5 seconds on first view" test.

**Cutlist FROM → room tab jump** — direct linkage from a cut piece back to its source room closes a real workflow loop. Combined with the chip label fix (room name instead of raw section ID), the cutlist now reads as a document, not a data dump.

### Priority Issues

**[P3] FROM chips have no static visual affordance for interactivity**
- **Why it matters**: On a tablet in a workshop, hover state and cursor changes don't exist before touch. Clickable and non-clickable chips share identical resting visual style (border, text-muted color). Users who haven't discovered the jump will never find it.
- **Fix**: Add a faint `↗` icon (or similar) after the label text on clickable chips only, and give button chips a subtle `background: rgba(212,145,58,0.05)` at rest. The hover state already becomes amber; just make the resting state differentiate interactive from static.
- **Command**: `/impeccable polish`

**[P3] Scenario switch gives no persistent hint that units/quantities also change**
- **Why it matters**: The scenario hint explains this on first use (and dismisses permanently). After dismissal, a user who returns to an old job with multiple options has no cue that clicking Option B will also replace their active unit quantities, not just the pricing summary. Risk of silent surprise.
- **Fix**: Add `title="Option B — switching loads its own unit quantities, materials and extras"` to each scenario tab. Or a static one-line caption below the strip (only when >1 scenario): `"Each option has its own unit quantities and extras."` — permanently visible, one line, no dismiss.
- **Command**: `/impeccable clarify`

**[P3] Duplicate job has no success toast**
- **Why it matters**: DUPLICATE closes the menu and switches to the new job. The switch itself is feedback, but the name change (`[original] (Copy)`) could be missed if the user isn't looking at the job name in the picker trigger. No confirmation that the action succeeded.
- **Fix**: Show a 2-second amber toast: `✓ Duplicated as '[Name] (Copy)'`. Same pattern as the autosave chip's confirmation pulse.
- **Command**: `/impeccable polish`

**[P3] ignore.md line-number references stale; 21 radius FPs uncatalogued**
- **Why it matters**: All line-number prose in ignore.md shifted ~100–250 lines since onboarding/runner/scenario code additions. The description-based rules still apply, but the comments mislead future maintainers. Additionally, 21 `border-radius: 2px/3px` occurrences in JS template strings still generate advisory hits not covered by ignore.md rules.
- **Fix**: Update line-number prose to "match by context, not line number" (description-based). Add a rule for `border-radius: 2px/3px in JS template literals` covering the cabinet dimension badge pills, icon chips, and picker row chips.
- **Command**: `/impeccable polish`

### Persona Red Flags

**Nico (SA tradesperson — daily user)**: Switches options fluently after reading the scenario hint. The "Option A → Option B → back to A" cycle works as expected — quantities are right. Remaining gap: on second visit to a saved job with multiple options, Nico sees the option tabs but no reminder that switching replaces unit quantities. If he switches by accident, he may not notice until he checks the cutlist.

**Alex (Power User)**: Finds ⧉ DUPLICATE immediately, uses it to spin up variant pricing. Tries keyboard shortcut for switching between Option A and B — none exists. Alt+1/Alt+2/Alt+3 for scenario tabs would be a natural extension of the existing Alt+1–5 navigation pattern. Not a blocker, but a missed efficiency for a power user.

**Jordan (First-Timer)**: Well-served by the scenario hint. Reads it fully, understands "unit quantities, extras, tops and custom panels." Clicks "+ Add option," sees it pre-fill. Only risk: if Jordan dismisses the hint within the first second without reading, it's gone permanently. No re-show mechanism (`scenHintDismissed` localStorage key set on dismiss). Consider a `?` button on the scenario strip to re-surface the hint.

**Sam (Accessibility)**: Scenario strip uses `role="tablist"` + `aria-selected` correctly. Rename and delete icons have `aria-label`. FROM chips use `title` for tooltip. Runner clearance preset buttons have `title` attributes but no explicit `aria-label` — on keyboards, title is read, so functional but not ideal. No regression vs run 8.

### Minor Observations

- `prompt()` used for scenario rename — native browser dialog breaks dark-theme immersion. In-place tab rename (double-click to edit) would feel more native. Not urgent.
- Runner clearance "Custom" → `runnerClearance=0` → then input takes over. If user clicks Custom then immediately clicks Standard, the flow is clean. But if user enters 0 in the custom field, clearance is 0 — no validation guard.
- The `rgba(212,168,67,0.07)` info box in runner clearance is close to amber family but not in DESIGN.md. Advisory-level; one line to register.
- `window.scrollTo({ top: 0, behavior: 'smooth' })` in `_cutlistJumpTo` is a good touch — but smooth scroll doesn't fire on `prefers-reduced-motion: reduce`. Minor.
