### Priority Issues

**[P3] Coach marks dismiss too early — clicking job picker closes the tour**
- `_openJobPicker()` calls `_closeOnboardCoachMark()`. Step 1 instructs "click here to load a job" — a first-timer does exactly that and loses steps 2 and 3.
- Fix: Remove `_closeOnboardCoachMark()` from `_openJobPicker()`. Only dismiss on `_selectJobFromPicker()` completion, SKIP, or GET STARTED.
- Command: `/impeccable polish`

**[P3] More hint collides with coach marks on first login**
- `showMoreHintOnce()` fires at 1500ms, coach marks at 900ms. On first login both activate within 600ms — two simultaneous "look at this!" affordances, different UI targets.
- Fix: In `showMoreHintOnce()` add early return: `if (!localStorage.getItem(_ONBOARD_KEY)) return;` — defer More hint until after onboarding complete.
- Command: `/impeccable polish`

**[P3] Coach mark tip card not keyboard-navigable**
- SKIP and NEXT buttons have no keyboard shortcut. Focus not moved to tip card on mount. Escape does nothing.
- Fix: On tip mount call `tip.focus()` (add `tabindex="-1"`). Add `keydown` listener: Escape = `_dismissOnboarding()`, → or Enter = next step.
- Command: `/impeccable polish`

**[P3] No `prefers-reduced-motion` for `onboardFadeIn`**
- `@keyframes onboardFadeIn` has no reduced-motion counterpart.
- Fix: `@media (prefers-reduced-motion: reduce) { #onboard-tip { animation: none; } }`
- Command: `/impeccable polish`

**[P3] `📐 PLAN [BETA]` has equal visual weight to primary actions**
- [BETA] suffix adds noise; amber-tint styling competes with Save and New.
- Fix: De-emphasize — `color: var(--text-muted); opacity: 0.75;` and move label to `data-tip` tooltip, or shorten to `📐 PLAN` with tooltip `[BETA] Upload a floor plan PDF for AI room extraction`.
- Command: `/impeccable clarify`

**[P3] ignore.md line-number comments stale**
- All line number references shifted ~85–200 lines due to onboarding + overflow additions. Logic is description-based so rules still apply, but comments mislead.
- Also: 21 new 2px/3px radius FPs in JS template strings not yet in ignore.md.
- Fix: Update line-number prose and add a description rule for `border-radius: 2px/3px in JS template literals`.
- Command: `/impeccable polish`
