## Priority Issues

**[P1] Silent part-drop on empty name in `saveUnitParts()`**
Line 16272: `if (!name) return;` skips any row where the name field is blank. A user (especially Jordan on her first session) can fill in dimensions, miss the name field, click Save Parts, and lose that row with no feedback.
**Fix:** Before saving, check for rows with non-empty dimensions but blank name. Show a toast: "1 part skipped — name is required." Or highlight the offending row's name cell in red and block Save until resolved.

**[P2] Numeric inputs lack tooltips**
Labour hours input now has a title= — but margin %, delivery fee, diesel km, consult fee, and rate fields don't. First-timers have to guess what unit (%, R, km, hours) each expects.
**Fix:** Add `title="e.g. 15 for 15% markup"` (and equivalents) to the main quote addon fields. Low effort, measurable H10 improvement.

**[P3] No undo/warn for part editor deletions**
Deleting a row from the parts modal and clicking Save Parts is irreversible with no confirmation. Unlike archived units (which have an Undo toast), deleted parts are gone.
**Fix:** Add `confirmAction('Remove X parts?')` before Save if the row count decreased, OR add an Undo toast post-save that restores the previous parts array.
