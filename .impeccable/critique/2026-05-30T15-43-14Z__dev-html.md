## Priority Issues

**[P2] `twoFieldPrompt` (promptAddEdge) closes silently on blank-size submit**
When the user clicks ADD with the Size field empty, `promptAddEdge` receives `['', '...']`, the `if (!size) return;` guard fires, and nothing happens. The modal has already closed. No toast, no re-focus, no red border. User has to re-click the Add Edging button to try again without knowing what went wrong.
**Fix:** Add `okBtn.addEventListener('click', ...)` validation before calling `finish()`: if `inputs[0].value.trim()` is empty, set `inputs[0].style.borderColor='#e05252'`, `inputs[0].focus()`, and return without closing. Or add a caller-supplied `validate(values)` hook to `twoFieldPrompt`.

**[P3] `importState.label` not reset on Import-tab revisit when status is idle**
The P3 issue from the previous pass remains open. `importState.label` resets on Quote-tab activation (line 7329) and on import clear (line 21131), but the `section === '__import__'` branch in `switchTab()` (line 6792) does not reset it. If a user visits Import, parses a file, navigates away, navigates back without clearing, then discards the result, the label field retains the previous file name on the next fresh visit.
**Fix:** In the `section === '__import__'` branch add: `if (importState.status === 'idle') { importState.label = ''; }` before `renderImportView()`.

## Minor Observations

- Per-unit labour hours input (line 10917) has no `title` attribute explaining the unit ("hours of labour per cabinet"). The inline formula hint "hrs × qty × R/hr" provides partial context but a tooltip would make the input unambiguous on first encounter.
- `aria-controls` still not wired between main-nav tab buttons and panel IDs — `role="tab"` and `aria-selected` present but panel association missing for screen readers. Unchanged from previous pass.
- Return focus on modal close still not implemented — `confirmAction()`, `inputPrompt()`, `twoFieldPrompt()` do not return focus to the triggering element. Minor a11y gap for keyboard-only users. Unchanged from previous pass.
- `importState.label` reset inconsistency (P3 above) unchanged.
- Labour config persisted to `localStorage` + Supabase `profiles.templates` — state round-trip is solid. One edge case: if two browser tabs are open with different jobs and one changes `labourConfig`, the other tab's in-memory state diverges silently. Low frequency; accepted given single-tab usage pattern.
