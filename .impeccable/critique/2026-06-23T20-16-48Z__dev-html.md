## Overall Impression

CutList Pro has real design identity. The amber-on-dark-charcoal system reads as premium trade software, not SaaS. The data density is right for the domain. The three-font system is working. Quote Scenarios is a genuine UX differentiator. The price history log shows the kind of domain-aware thinking that earns trust from tradespeople.

The gaps live in discoverability and help infrastructure — features exist but are hard to find without prior knowledge, and new users (non-software-native tradespeople) have no onboarding, no empty-state guidance, and no undo path for the actions they're most likely to get wrong.

---

## What's Working

1. **Brand coherence.** Amber focus rings, active tabs, quantity highlights, and action buttons all speak the same language. The palette is controlled and consistent.
2. **JetBrains Mono for numbers.** Scanning columns and comparing figures is fast because numeric data is always monospaced. The Mono Rule is holding.
3. **Quote Scenarios.** Generating A/B/C options from a single job is a standout power feature with clear UX thought behind it.
4. **Price change history log.** A thoughtful audit trail that tradespeople will use. Exactly the right kind of domain-specific feature.
5. **Amber qty highlight.** The amber-tinted input for non-zero quantities is subtle, effective, and on-brand.
6. **Reduced motion support.** `@media (prefers-reduced-motion: reduce)` at line 3013 shows accessibility was considered for animations.
7. **Contrast ratios.** `--text (#e8e0d0)` on `--bg (#141414)` ≈ 13:1. `--text-muted (#9c8470)` on `--bg` ≈ 5:1. Both pass AA. Dark theme is solid.

## Priority Issues

### P1 — Side-tab border ban violated (P&L invoice grouping)

**Lines 26564 and 26586** — `border-left: 4px solid var(--amber)` on the collapsible invoice document header and its expanded line container.

The hierarchy intent is legitimate: grouping imported invoice lines under a document header. The side-stripe execution is the absolute ban. The amber chevron + amber invoice number on that same row are already doing the hierarchy work — the border is redundant.

**Fix options (pick one):**
- Replace with a tinted background: `background: rgba(amber-rgb, 0.06)` on the header row
- Or replace with a full border: `border: 1px solid var(--border)` + slightly elevated background on the row
- Or drop the left border entirely — the chevron + indentation of child lines already shows the nesting

### P1 — Help and Documentation gap

**Score: 1/4.** Icon-only action buttons (⚙ Special Services, ⧉ Duplicate, × Remove) have `title` attributes — visible on desktop hover, invisible on tablet touchscreens, not announced by screen readers. New tradespeople have no onboarding, no empty states with guidance, and no recoverable undo for panel edits.

**Immediate wins (no full onboarding flow required):**
- Add `aria-label` to all `.is-icon` buttons
- Add a "No panels yet — add your first panel above" empty state to the Panels tab
- Consider a short visible label under the three panel action icons on mobile widths (or a persistent legend row)

---

### P2 — Recognition over Recall

**Edge notation dependency.** L1/L2/W1/W2 makes sense once learned, but the info button is small and easy to miss. A persistent collapsed legend row above the panel table (state remembered in localStorage) would eliminate the recall dependency without adding noise.

**"More" overflow tab.** Features in the overflow dropdown are invisible to new users. A small visual indicator ("and 3 more") or a first-run hint would help without restructuring the nav.

### P2 — Error Recovery gap

No undo for panel row changes (material, dimensions, edging). In a pricing context, a wrong material selection can silently distort the quote total. Even a single-level "revert this panel to last saved state" would cover the main failure mode.

---

### P3 — Color registry gaps in DESIGN.md

Detector found many literal hex values in dev.html not registered in DESIGN.md. Most are expected (DESIGN.md is new), but two categories warrant formal registration:

- **Off-cut map colors** (`#66aa88`, `#aaaa66`, `#cc7766`) — these are semantic: they mean something specific in the sheet optimizer (material type / cut type identification). A `functional-palette` section in DESIGN.md would prevent future editors from treating them as arbitrary hex. Also flag for colorblind safety — the green vs red-orange distinction is a deuteranopia risk if the optimizer is used as a workshop picking guide.
- **Gradient stop** `#b8852a` — register as `--amber-gradient-mid` or add an inline comment marking it intentional.

### P3 — Broken image console errors

`<img src="">` on `fb-screenshot-preview` (line 4710) and `img-lightbox-img` (line 4888) generate a failed network request on every page load. Replace with `src="data:,"` or omit the `src` attribute entirely and set it only when the lightbox opens via JS.

## Persona Red Flags

**Alex (Power User — experienced cabinet maker, 10+ quotes/week)**
- No keyboard shortcuts for common actions (add panel, duplicate row, tab navigation). Mouse-only workflow.
- No batch operations on panel rows — changing material for 20 panels requires 20 individual selects.
- "More" dropdown requires a mouse click each session — no way to pin secondary tabs to the main bar.

**Sam (Accessibility-Dependent — tablet + screen reader, workshop context)**
- ⚙ ⧉ × icon buttons: `title` attributes exist but no `aria-label`. Screen readers may announce these with no meaningful label.
- Amber focus rings implemented (`button:focus-visible`) — good.
- Reduced motion handled at line 3013 — good.
- Contrast on dark theme passes AA — good.
- The one contextual help element (L1/L2/W1/W2 info button) is a `<button>` — accessible. But it's the only one.

**Nico (Owner-Operator — manages supplier pricing, sends quotes)**
- Switching between Supplier Pricing and individual job material overrides requires remembering original values — no side-by-side diff view.
- Quote Scenarios output is strong, but no internal "scenario diff" shows Nico quickly what changed between options A, B, and C.
- The P&L tab's invoice grouping (where both side-tab violations live) is directly in Nico's critical workflow path.

## Questions to Consider

1. **Undo depth.** Would "revert panel to last saved state" be enough, or do users need multi-step undo within a session? The answer drives implementation complexity significantly.
2. **P&L side-tab fix.** The tinted-background approach changes the visual weight of the invoice grouping. Worth a quick visual check: does collapse/expand still read clearly without the left border as a vertical continuation line?
3. **Onboarding priority.** Is the current growth path new-user acquisition or power-user retention? That determines whether the Help/Documentation P1 moves to fix-now or fix-next-quarter.
4. **Off-cut map colorblind safety.** Is the sheet optimizer used as a physical cutting guide in the workshop? If yes, the green/red-orange distinction needs a shape or pattern alternative alongside color.
